# セキュリティポリシー

## 想定する脅威モデル

MyHome Vision は、住宅購入検討客の **顔を含む家族写真** を扱うサービスです。以下を最重要リスクとして扱います。

1. 生体情報・顔写真・氏名・住所等の **個人識別情報（PII）の漏えい**
2. 同意レベル（L1 国内 / L2 US 許可 / L3 全許可）を **超えたプロバイダーへの送信**
3. 生成済み動画の **権限外アクセス / 外部流出**
4. 第三者プロバイダー経由の **データ再利用・学習データ化**
5. `failed_generations` ログ等 **運用データからの再特定攻撃**

## 基本方針

- 平文PIIは **DBにもログにも残さない**（`backend/app/core/anonymization.py` のSHA256+salt を経由）
- 生成失敗ログ（`failed_generations`）は **匿名化後2年保持**、それ以降は自動削除
- 外部プロバイダーへの送信は **同意レベルごとにゲート**（`backend/app/providers/registry.py`）
- 新規プロバイダー連携は **デフォルトOFF（Dark Launch）** で投入（`KLING_ENABLED=false` が既定）
- 秘密情報は `.env` にのみ置き、**コミットしない**（`.env.example` のみをリポジトリに含める）

## 脆弱性の報告方法

**公開 Issue には書かないでください。**

発見した脆弱性・プライバシー関連の懸念は、以下のいずれかで **非公開** に報告してください。

- リポジトリオーナーへの DM（sisuiss）
- メール: kizuku113@gmail.com（件名に `[SECURITY] MyHome Vision` を付けてください）
- GitHub の Private Vulnerability Reporting（本リポジトリの Security タブ）

報告には以下を含めてください：
- 影響を受ける箇所（ファイル / エンドポイント / ブランチ）
- 再現手順
- 想定される影響範囲（PII漏えい / 権限昇格 / DoS 等）
- 可能であれば PoC

## 対応フロー（目安）

| ステップ | 期限目安 |
|---------|---------|
| 初回返信 | 3営業日以内 |
| 影響範囲の確定 | 7営業日以内 |
| 修正方針の共有 | 14営業日以内 |
| 修正リリース | 重大度に応じて、最短24時間〜30日 |

## スコープ外

- 本番稼働していないステージング環境でのみ再現する軽微な不具合
- 一般的な DoS（大量リクエストによる負荷試験）
- 依存ライブラリのCVEのうち、実際には本プロダクトで到達不能なコードパス
